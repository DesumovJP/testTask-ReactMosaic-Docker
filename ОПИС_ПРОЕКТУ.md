# Опис проекту: Imagino Dashboard

## 📋 Загальний огляд

**Imagino Dashboard** — це веб-додаток для відображення інформації про компанії у вигляді інтерактивного дашборду з трьома переміщуваними та змінюваними за розміром віджетами. Проект реалізований як тестове завдання з використанням сучасного React-стеку та TypeScript.

## 🎯 Основні функції

### 1. Інтерактивний дашборд
- **Три віджети** з інформацією про компанії, розташовані в мозаїчному layout
- **Зміна розміру панелей** — користувач може змінювати розмір кожного віджета перетягуванням меж
- **Переміщення панелей** — можливість змінювати розташування віджетів drag-and-drop
- **Адаптивний дизайн** — коректне відображення на різних розмірах екранів

### 2. Відображення даних про компанії
Кожен віджет показує детальну інформацію про вибрану компанію:
- Базові дані: ticker, назва, юридична назва
- Фінансова інформація: біржа, сектор, галузь
- Контактні дані: адреса, телефон, веб-сайт
- Додаткова інформація: кількість співробітників, дати філінгів, описи

### 3. Вибір компанії
- **Dropdown меню** в кожному віджеті для вибору компанії за тикером
- **Незалежний вибір** — кожен віджет може відображати різну компанію
- **Автоматичне завантаження** списку доступних компаній при старті

### 4. Темна/Світла тема
- **Три теми**: Blueprint (за замовчуванням), Dark, Light
- **Перемикання теми** через dropdown в хедері
- **Адаптація всіх компонентів** під вибрану тему

### 5. Обробка помилок
- **Стан завантаження** — відображення спінера під час запитів
- **Повідомлення про помилки** з можливістю повторного запиту
- **Валідація даних** — перевірка наявності даних перед відображенням

### Потік даних

1. **Запуск додатку**
   - Користувач відкриває додаток у браузері
   - React завантажується та рендерить компонент `Dashboard`
   - Виконується запит `GET /api/companies` для отримання списку компаній

2. **Ініціалізація Mosaic Layout**
   - Компонент `Mosaic` створює три панелі згідно з `initialValue`
   - Кожна панель обгортається в `MosaicWindow`
   - Всередині кожної панелі рендериться `CompanyInfoWidget`

3. **Завантаження даних**
   - Кожен віджет має свій `selectedTicker` (за замовчуванням: 'AAPL', 'AAPL', 'NVDA')
   - При монтуванні віджет робить запит `GET /api/companies/:ticker`
   - Під час завантаження відображається `Spinner`
   - Після отримання даних відображається інформація про компанію

4. **Зміна вибраної компанії**
   - Користувач вибирає новий тикер з dropdown
   - Викликається `onTickerChange`, який оновлює `selectedTickers` в `Dashboard`
   - Віджет отримує новий `ticker` prop
   - `useEffect` в `CompanyInfoWidget` спрацьовує на зміну `ticker`
   - Виконується новий API запит
   - Оновлюються дані в віджеті

5. **Зміна розміру/позиції панелей**
   - Користувач перетягує межі між панелями
   - React Mosaic автоматично оновлює розміри
   - Стан layout зберігається в React Mosaic (не зберігається між сесіями)

## 🚀 Запуск та розгортання

1. **Встановлення залежностей**
   ```bash
   yarn install
   ```

2. **Запуск API сервера** (термінал 1)
   ```bash
   yarn server
   ```
   Сервер запуститься на `http://localhost:3001`

3. **Запуск dev-сервера** (термінал 2)
   ```bash
   yarn dev
   ```
   Додаток буде доступний на `http://localhost:5173`

1. **Збірка проекту**
   ```bash
   yarn build
   ```
   Створюється папка `dist` з оптимізованими файлами

2. **Перегляд production версії**
   ```bash
   yarn preview
   ```

### Docker розгортання

1. **Збірка Docker образу**
   ```bash
   docker build -t imagino-dashboard .
   ```

2. **Запуск контейнера**
   ```bash
   docker run -p 3001:3001 imagino-dashboard
   ```

3. **Доступ до додатку**
   Відкрити `http://localhost:3001` у браузері

## 📡 API Endpoints

### GET /api/companies
Повертає список всіх доступних компаній.

**Відповідь:**
```json
[
  {
    "id": "com_NX6GzO",
    "ticker": "AAPL",
    "name": "Apple Inc.",
    ...
  }
]
```

### GET /api/companies/:ticker
Повертає детальну інформацію про конкретну компанію.

**Параметри:**
- `ticker` (string) — тикер компанії (наприклад, "AAPL")

**Відповідь:**
```json
{
  "id": "com_NX6GzO",
  "ticker": "AAPL",
  "name": "Apple Inc.",
  "legal_name": "Apple Inc.",
  "stock_exchange": "Nasdaq",
  ...
}
```

**Помилки:**
- `404` — компанія не знайдена
- `500` — помилка сервера

## 🔧 Технічні деталі реалізації

### React Mosaic Integration

React Mosaic використовується для створення гнучкого layout з панелями:

```typescript
const initialValue = {
  direction: 'row',
  first: '1',
  second: {
    direction: 'column',
    first: '2',
    second: '3',
  },
};
```

Це створює layout:
```
┌─────────┬─────────┐
│         │    2    │
│    1    ├─────────┤
│         │    3    │
└─────────┴─────────┘
```

### State Management

Стан додатку управляється через React hooks:
- `useState` — локальний стан компонентів
- `useEffect` — побічні ефекти (API запити)
- Підйом стану (`lifted state`) — `selectedTickers` в `Dashboard`

### TypeScript Типізація

Всі дані типізовані через інтерфейси:
- `Company` — структура даних компанії
- `CompanyInfoWidgetProps` — пропси для віджетів
- `ViewId` — тип для ID панелей Mosaic

### Стилізація

- **Tailwind CSS** — utility класи для швидкої стилізації
- **CSS Modules** — кастомні стилі для Mosaic
- **Умовні класи** — динамічна зміна стилів залежно від теми
